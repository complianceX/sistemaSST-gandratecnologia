## Objetivo
- colocar o browser do Puppeteer dentro do artefato do backend no Render runtime Node
- parar de depender de cache fora do app para geração de PDF
- melhorar o diagnóstico de falha do Chromium em produção

## Mudanças
- adiciona `backend/.puppeteerrc.cjs` com cache local em `backend/.cache/puppeteer`
- adiciona `backend/scripts/ensure-puppeteer-browser.cjs` no `postinstall`
- resolve `executablePath` via `puppeteer.executablePath()` quando a env não vier setada
- endurece logs de falha do launch do Chromium
- adiciona cobertura unitária do fallback de `executablePath`

## Validação
- `node scripts/ensure-puppeteer-browser.cjs`
- `npm test -- --runInBand --testPathPatterns="src/common/services/puppeteer-pool.service.spec.ts|src/common/services/pdf.service.spec.ts|src/aprs/services/aprs-pdf.service.spec.ts"`
- `npx eslint --no-warn-ignored src/common/services/puppeteer-pool.service.ts src/common/services/puppeteer-pool.service.spec.ts`
